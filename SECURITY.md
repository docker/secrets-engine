# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in this project, please report it
responsibly. **Do not open a public GitHub issue.**

You can report security issues by:

- Emailing [security@docker.com](mailto:security@docker.com)
- Using [GitHub's private vulnerability reporting](https://github.com/docker/secrets-engine/security/advisories/new)

Please include:

- A description of the vulnerability
- Steps to reproduce the issue
- Any relevant logs or screenshots
- The potential impact of the vulnerability

## What Qualifies as a Security Issue

- Authentication or authorization bypasses
- Secret leakage or exposure
- Injection vulnerabilities
- Privilege escalation
- Cryptographic weaknesses

## Response Timeline

- We will acknowledge receipt of your report within **3 business days**.
- We will provide an initial assessment within **10 business days**.
- We aim to release a fix for critical vulnerabilities as quickly as possible.

## Supported Versions

Security updates are applied to the latest release. We recommend always running
the latest version.
